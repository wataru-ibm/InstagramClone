### ZoomTransitionDelegate.swift

```swift
class ZoomTransitionDelegate: NSObject, UIViewControllerTransitioningDelegate {
    private let zoomAnimator = ZoomAnimator()
    var fromImageView: UIImageView?

    func animationController(forPresented presented: UIViewController, presenting: UIViewController, source: UIViewController) -> (any UIViewControllerAnimatedTransitioning)? {
        zoomAnimator.isPresenting = true
        zoomAnimator.fromImageView = fromImageView
        return zoomAnimator
    }

    func animationController(forDismissed dismissed: UIViewController) -> (any UIViewControllerAnimatedTransitioning)? {
        zoomAnimator.isPresenting = false
        return zoomAnimator
    }
}
```

### SearchViewController.swift（追加・変更箇所）

// クラス本体のプロパティに追加
private let zoomTransitionDelegate = ZoomTransitionDelegate()

extension SearchViewController: UICollectionViewDelegate {

    }
}

## Week3 Step4: ZoomAnimator アニメーション実装（CGAffineTransform）

### なぜ frame ではなく CGAffineTransform を使うのか

`frame` を直接アニメーションで変更すると Auto Layout と競合してズレが生じる。`CGAffineTransform` は「見た目だけを変形する」ものなので Auto Layout と干渉しない。

```
frame    → 実際の居場所（変えない）
transform → 見た目だけ変形（アニメーションで戻す）
```

### アニメーションの考え方

```
① toVC.view をフルスクリーンの位置に置く（frame = finalFrame）
② transform でサムネイルに見えるよう変装（縮小 + 位置ずらし）
③ alpha = 0 で透明にする
④ UIView.animate で transform → .identity, alpha → 1
   → フルスクリーンにズームインしながらフェードイン
```

### ZoomAnimator.swift（完成版）

```swift
func animateTransition(using transitionContext: UIViewControllerContextTransitioning) {
    guard let toVC = transitionContext.viewController(forKey: .to) else { return }
    guard let fromVC = transitionContext.viewController(forKey: .from) else { return }
    let containerView = transitionContext.containerView

    if isPresenting {
        containerView.addSubview(toVC.view)
        guard let fromImageView = fromImageView else { return }

        let startFrame = fromImageView.convert(fromImageView.bounds, to: containerView)
        let finalFrame = transitionContext.finalFrame(for: toVC)
        toVC.view.frame = finalFrame

        let scaleX = startFrame.width / finalFrame.width
        let scaleY = startFrame.height / finalFrame.height
        let tx = startFrame.midX - finalFrame.midX
        let ty = startFrame.midY - finalFrame.midY

        toVC.view.transform = CGAffineTransform(scaleX: scaleX, y: scaleY)
            .concatenating(CGAffineTransform(translationX: tx, y: ty))
        toVC.view.alpha = 0

        UIView.animate(withDuration: transitionDuration(using: transitionContext), animations: {
            toVC.view.transform = .identity
            toVC.view.alpha = 1
        }, completion: { _ in
            transitionContext.completeTransition(!transitionContext.transitionWasCancelled)
        })

    } else {
        guard let fromImageView = fromImageView else { return }

        let startFrame = fromImageView.convert(fromImageView.bounds, to: containerView)
        let scaleX = startFrame.width / fromVC.view.frame.width
        let scaleY = startFrame.height / fromVC.view.frame.height
        let tx = startFrame.midX - fromVC.view.frame.midX
        let ty = startFrame.midY - fromVC.view.frame.midY

        UIView.animate(withDuration: transitionDuration(using: transitionContext), animations: {
            fromVC.view.transform = CGAffineTransform(scaleX: scaleX, y: scaleY)
                .concatenating(CGAffineTransform(translationX: tx, y: ty))
            fromVC.view.alpha = 0
        }, completion: { _ in
            fromVC.view.transform = .identity
            transitionContext.completeTransition(!transitionContext.transitionWasCancelled)
        })
    }
}
```

### 各変数の意味

| 変数 | 意味 |
|---|---|
| `startFrame` | サムネイル画像の containerView 上での位置・サイズ |
| `finalFrame` | PostDetail のフルスクリーン位置・サイズ（UIKit が返す） |
| `scaleX / scaleY` | サムネイル ÷ フルスクリーン の縮小比率 |
| `tx / ty` | サムネイルの中心とフルスクリーンの中心のズレ |

## Q&A

### UIViewControllerTransitioningDelegate の2つのメソッドはなぜ必要？

通常の `present` はUIKitがデフォルトのフェードアニメーションを使う。カスタムアニメーションを差し込むには `transitioningDelegate` を通じてUIKitに「このアニメーションは誰が担当する？」と教える必要がある。

```
present する → animationController(forPresented:) が呼ばれる → アニメーターを返す
dismiss する → animationController(forDismissed:)  が呼ばれる → アニメーターを返す
```

`nil` を返すとデフォルト（フェード）にフォールバック。オブジェクトを返すとそのオブジェクトの `animateTransition` が呼ばれる。

---

### animationController(forPresented:) の3つの引数は何？

`present(detailVC, animated: true)` を呼んだとき：

| 引数 | 意味 | 今回 |
|---|---|---|
| `presented` | これから表示されるVC | PostDetailViewController |
| `presenting` | 表示する側のVC | SearchViewController |
| `source` | presentを呼び出したVC | 基本的にpresentingと同じ |

**今回のコードでは3つとも使わない。** `return zoomAnimator` だけでOK。

---

### dismiss で fromImageView を設定しない理由は？

`forPresented` のときにすでに `zoomAnimator.fromImageView = fromImageView` を設定している。`zoomAnimator` は `private let` で**同じインスタンス**を持ち続けるので、dismiss のときも値がそのまま残っている。再代入が不要。

```
present時:  zoomAnimator.fromImageView = セルのimageView  ← ここで代入
dismiss時:  zoomAnimator.fromImageView → まだセルのimageViewが入ったまま
```

---

### ZoomTransitionDelegate になぜ isPresenting / isDismissing プロパティは不要？

このフラグは `ZoomAnimator` がすでに持っている。`ZoomTransitionDelegate` の役割は「状態を持つ」ではなく「アニメーターを返す係」。状態（isPresenting）は各メソッドの中で `ZoomAnimator` に設定して渡す。

---

### collectionView.delegate = self の意味は？

UICollectionViewはイベント（セルタップなど）が起きたとき、誰かに教える相手が必要。`delegate = self` でそれを指定する。

これを書かないと `didSelectItemAt` を実装しても**一切呼ばれない**。

| プロパティ | 役割 |
|---|---|
| `dataSource` | データは誰が持ってる？ |
| `delegate` | イベントは誰に教える？ |

---

### extension に UICollectionViewDelegate を書く理由は？

`collectionView.delegate = self` の `self` を代入するには、`SearchViewController` が `UICollectionViewDelegate` に準拠している宣言が必要。宣言がないとコンパイラが「このクラスはそのプロトコルを実装してない」とエラーを出す。

extension で書くのは整理のため（クラス宣言に直接書いても動く）。

---

### guard let と if let の違いは？

| | データあり | データなし |
|---|---|---|
| `if let` | `{}` の中に入る | スキップ |
| `guard let` | そのまま続行 | `else { return }` で終了 |

`guard let` は「ない場合はここで終わり、以降のコードはある前提で書く」という早期リターンのパターン。

---

### toVC / fromVC は present と dismiss で逆転する

| | `.from` | `.to` |
|---|---|---|
| present（表示） | SearchViewController | PostDetailViewController |
| dismiss（戻る） | PostDetailViewController | SearchViewController |

ZoomAnimator の dismiss のコードが `fromVC.view.frame` を縮小させているのはこの逆転を利用している。

---

### SearchGridCell の thumbnailImageView はなぜ計算プロパティ？

```swift
var thumbnailImageView: UIImageView { postImageView }
```

`postImageView` 自体は `private` のままにしつつ、外からは読み取り専用で公開するため。`private` を外すより意図が明確になる。

---

### modalPresentationStyle = .fullScreen はなぜ必要？

iOS13以降のデフォルトはカード型（下からシュッと出るスタイル）になる。これだとズームアニメーションと相性が悪いため `.fullScreen` を指定する。

---

### containerView とは？

トランジション専用の一時的な親 View。`from` と `to` の両方の View をここに乗せてアニメーションする。トランジションが終わると UIKit が後片付けをしてくれる。

`containerView.addSubview(toVC.view)` を書かないと `toVC.view` は画面に一切表示されない。カスタムトランジションでは UIKit が自動でやってくれる処理を自分でやる必要がある。

---

### .convert(fromImageView.bounds, to: containerView) で bounds を渡す理由は？

`fromImageView` は CollectionViewCell の中にいるため、`.frame` は「Cell の中での位置」になる。`.convert(_:to:)` は「変換元の View の座標系」で渡した rect を受け取るので、fromImageView 自身の座標系（= bounds）を渡すことで「containerView 上のどこにあるか」に変換できる。

| プロパティ | 座標系 |
|---|---|
| `fromImageView.bounds` | fromImageView 自身（0, 0 始まり） |
| `fromImageView.frame` | fromImageView の親（Cell） |
| `.convert(bounds, to: containerView)` の結果 | containerView 基準 |

---

### transitionContext とは？

UIKit からの「今回のトランジションに必要な情報はここから全部取ってください」という手渡しパッケージ。

| メソッド / プロパティ | 返すもの |
|---|---|
| `.viewController(forKey: .from)` | 遷移元の VC |
| `.viewController(forKey: .to)` | 遷移先の VC |
| `.containerView` | アニメーションの舞台 |
| `.finalFrame(for: toVC)` | toVC の最終的な frame |

---

### .concatenating と .translatedBy の違いは？

`.translatedBy` は「scale がかかった後の座標系」で translate するため、tx・ty の値をスケール分で割り戻す計算が必要になる。`.concatenating` はスクリーン座標系で素直に tx・ty をそのまま使える。

```swift
// concatenating を使う場合（tx/ty はスクリーン座標のまま）
CGAffineTransform(scaleX: scaleX, y: scaleY)
    .concatenating(CGAffineTransform(translationX: tx, y: ty))

// translatedBy を使う場合（割り戻しが必要）
CGAffineTransform(scaleX: scaleX, y: scaleY)
    .translatedBy(x: tx / scaleX, y: ty / scaleY)
```

---

### completion で fromVC.view.transform = .identity が必要な理由は？

dismiss 完了後に `fromVC.view`（PostDetail）は画面から消えるが、transform が縮小状態のまま残る。次回 present したときに縮んだ状態で登場してしまうので、後片付けとして `.identity` にリセットする。
