### 全体の流れ

```
① ユーザーがグリッドのセルをタップ
② didSelectItemAt が呼ばれる（SearchViewController）
③ UIKit が ZoomTransitionDelegate の animationController(forPresented:) を呼ぶ
④ ZoomTransitionDelegate が ZoomAnimator を返す
⑤ UIKit が ZoomAnimator.animateTransition を呼ぶ
⑥ ズームアニメーションで PostDetailViewController が表示される
⑦ 画面タップで dismiss → ZoomAnimator の dismiss側アニメーションが動く
```

### ZoomTransitionDelegate.swift

```swift
class ZoomTransitionDelegate: NSObject, UIViewControllerTransitioningDelegate {
    private let zoomAnimator = ZoomAnimator()
    var fromImageView: UIImageView?

    func animationController(forPresented presented: UIViewController, presenting: UIViewController, source: UIViewController) -> (any UIViewControllerAnimatedTransitioning)? {
        zoomAnimator.isPresenting = true
        zoomAnimator.fromImageView = fromImageView
        return zoomAnimator
    }

    func animationController(forDismissed dismissed: UIViewController) -> (any UIViewControllerAnimatedTransitioning)? {
        zoomAnimator.isPresenting = false
        return zoomAnimator
    }
}
```

### SearchViewController.swift（追加・変更箇所）

```swift
// クラス本体のプロパティに追加
private let zoomTransitionDelegate = ZoomTransitionDelegate()

// setupUI() に追加
collectionView.delegate = self

// extension
extension SearchViewController: UICollectionViewDelegate {
    func collectionView(_ collectionView: UICollectionView, didSelectItemAt indexPath: IndexPath) {
        guard let post = dataSource.itemIdentifier(for: indexPath) else { return }
        guard let cell = collectionView.cellForItem(at: indexPath) as? SearchGridCell else { return }
        zoomTransitionDelegate.fromImageView = cell.thumbnailImageView

        let detailVC = PostDetailViewController(post: post)
        detailVC.transitioningDelegate = zoomTransitionDelegate
        detailVC.modalPresentationStyle = .fullScreen
        present(detailVC, animated: true)
    }
}
```

## Q&A

### UIViewControllerTransitioningDelegate の2つのメソッドはなぜ必要？

通常の `present` はUIKitがデフォルトのフェードアニメーションを使う。カスタムアニメーションを差し込むには `transitioningDelegate` を通じてUIKitに「このアニメーションは誰が担当する？」と教える必要がある。

```
present する → animationController(forPresented:) が呼ばれる → アニメーターを返す
dismiss する → animationController(forDismissed:)  が呼ばれる → アニメーターを返す
```

`nil` を返すとデフォルト（フェード）にフォールバック。オブジェクトを返すとそのオブジェクトの `animateTransition` が呼ばれる。

---

### animationController(forPresented:) の3つの引数は何？

`present(detailVC, animated: true)` を呼んだとき：

| 引数 | 意味 | 今回 |
|---|---|---|
| `presented` | これから表示されるVC | PostDetailViewController |
| `presenting` | 表示する側のVC | SearchViewController |
| `source` | presentを呼び出したVC | 基本的にpresentingと同じ |

**今回のコードでは3つとも使わない。** `return zoomAnimator` だけでOK。

---

### dismiss で fromImageView を設定しない理由は？

`forPresented` のときにすでに `zoomAnimator.fromImageView = fromImageView` を設定している。`zoomAnimator` は `private let` で**同じインスタンス**を持ち続けるので、dismiss のときも値がそのまま残っている。再代入が不要。

```
present時:  zoomAnimator.fromImageView = セルのimageView  ← ここで代入
dismiss時:  zoomAnimator.fromImageView → まだセルのimageViewが入ったまま
```

---

### ZoomTransitionDelegate になぜ isPresenting / isDismissing プロパティは不要？

このフラグは `ZoomAnimator` がすでに持っている。`ZoomTransitionDelegate` の役割は「状態を持つ」ではなく「アニメーターを返す係」。状態（isPresenting）は各メソッドの中で `ZoomAnimator` に設定して渡す。

---

### collectionView.delegate = self の意味は？

UICollectionViewはイベント（セルタップなど）が起きたとき、誰かに教える相手が必要。`delegate = self` でそれを指定する。

これを書かないと `didSelectItemAt` を実装しても**一切呼ばれない**。

| プロパティ | 役割 |
|---|---|
| `dataSource` | データは誰が持ってる？ |
| `delegate` | イベントは誰に教える？ |

---

### extension に UICollectionViewDelegate を書く理由は？

`collectionView.delegate = self` の `self` を代入するには、`SearchViewController` が `UICollectionViewDelegate` に準拠している宣言が必要。宣言がないとコンパイラが「このクラスはそのプロトコルを実装してない」とエラーを出す。

extension で書くのは整理のため（クラス宣言に直接書いても動く）。

---

### guard let と if let の違いは？

| | データあり | データなし |
|---|---|---|
| `if let` | `{}` の中に入る | スキップ |
| `guard let` | そのまま続行 | `else { return }` で終了 |

`guard let` は「ない場合はここで終わり、以降のコードはある前提で書く」という早期リターンのパターン。

---

### toVC / fromVC は present と dismiss で逆転する

| | `.from` | `.to` |
|---|---|---|
| present（表示） | SearchViewController | PostDetailViewController |
| dismiss（戻る） | PostDetailViewController | SearchViewController |

ZoomAnimator の dismiss のコードが `fromVC.view.frame` を縮小させているのはこの逆転を利用している。

---

### SearchGridCell の thumbnailImageView はなぜ計算プロパティ？

```swift
var thumbnailImageView: UIImageView { postImageView }
```

`postImageView` 自体は `private` のままにしつつ、外からは読み取り専用で公開するため。`private` を外すより意図が明確になる。

---

### modalPresentationStyle = .fullScreen はなぜ必要？

iOS13以降のデフォルトはカード型（下からシュッと出るスタイル）になる。これだとズームアニメーションと相性が悪いため `.fullScreen` を指定する。
